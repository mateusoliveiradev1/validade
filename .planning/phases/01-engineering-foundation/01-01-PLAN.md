<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| package manager -> local workspace | External dependencies enter the repository during install. |
| shared packages -> future apps | Package exports become trusted contracts consumed by apps. |
| fake adapter -> provider adapters | Phase 1 fake behavior could be mistaken for production provider behavior later. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `package.json`, `pnpm-lock.yaml` | mitigate | Pin package manager, commit lockfile after install, and run `pnpm install --frozen-lockfile` in CI. |
| T-01-02 | Information Disclosure | shared package examples | mitigate | Use fictitious fixtures only and keep provider credentials out of package source. |
| T-01-03 | Elevation of Privilege | package dependency graph | mitigate | Add dependency-boundary checks in Plan 03 so domain cannot import app/provider layers. |
| T-01-SC | Tampering | npm installs | mitigate | Use lockfile, `pnpm audit --audit-level high`, and CI checks before accepting dependency changes. |
</threat_model>
