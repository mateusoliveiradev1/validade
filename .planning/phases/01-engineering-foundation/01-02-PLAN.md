<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser/mobile -> API | Untrusted local client requests reach Hono route handlers. |
| API -> fake adapter | API writes and reads safe probe state through adapter interface. |
| app config -> runtime | Public env names are visible to client apps. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `/probe` input | mitigate | Validate all probe payloads with Zod contracts and return 400 on invalid input. |
| T-02-02 | Information Disclosure | app/API responses | mitigate | Tests assert no secret-like strings and fixtures remain fictitious. |
| T-02-03 | Spoofing | API health result | accept | Health endpoint exposes only non-sensitive service/status metadata for local smoke. |
| T-02-04 | Elevation of Privilege | mobile/web direct data access | mitigate | Mobile/web consume API/contracts only; no privileged database credentials are introduced. |
</threat_model>
