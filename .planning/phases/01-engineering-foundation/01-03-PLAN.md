<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer machine -> public git repo | Local env files and generated artifacts could be committed. |
| source code -> runtime config | Env parsing decides which values apps/API trust. |
| package imports -> architectural tiers | Invalid imports can leak provider or UI concerns into protected packages. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | `.env*`, fixtures, docs | mitigate | `.gitignore`, secretlint, config tests, and data safety script block secrets/real data. |
| T-03-02 | Tampering | dependency boundaries | mitigate | `scripts/check-boundaries.mjs` fails on forbidden imports. |
| T-03-03 | Repudiation | quality gate output | mitigate | Root scripts give deterministic pass/fail signals for local and CI logs. |
| T-03-04 | Elevation of Privilege | client apps | mitigate | Boundary checks prevent mobile/web from importing privileged provider/database modules. |
</threat_model>
