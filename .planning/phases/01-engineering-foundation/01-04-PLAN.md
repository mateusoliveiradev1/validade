<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tests -> confidence | Weak tests can give false assurance about unbuilt behavior. |
| fixtures -> public repo | Example operational data can leak real store/customer context. |
| E2E tooling -> CI | Browser/emulator tools may fail if treated as mandatory without setup. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Repudiation | test results | mitigate | Root commands produce non-watch pass/fail logs. |
| T-04-02 | Information Disclosure | fixtures and evidence examples | mitigate | Fixture tests and data safety script require fictitious markers. |
| T-04-03 | Denial of Service | CI smoke runtime | mitigate | Keep `pnpm check` to CI-safe commands and document browser/emulator prerequisites separately. |
| T-04-04 | Tampering | mutation config | accept | Stryker config is present but mutation enforcement waits for critical rules; this is documented in `docs/testing/strategy.md`. |
</threat_model>
