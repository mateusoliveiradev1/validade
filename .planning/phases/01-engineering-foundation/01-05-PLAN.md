<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local repo -> GitHub Actions | CI runs untrusted pull-request changes against the quality suite. |
| documentation -> operator behavior | Incomplete docs can lead to secrets or real data being committed. |
| dependency updates -> runtime | New packages can introduce vulnerable code paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | CI workflows | mitigate | Workflows run frozen install and root checks; dependency review blocks high/critical vulnerable additions. |
| T-05-02 | Information Disclosure | README and examples | mitigate | Docs explicitly forbid secrets, real operational data, and evidence assets; scripts scan the repo. |
| T-05-03 | Repudiation | release readiness | mitigate | Threat model documents severity rules and high/critical blocking criteria. |
| T-05-04 | Denial of Service | free-tier CI/provider usage | mitigate | Free-pilot limits doc warns before heavy CI/live provider usage. |
</threat_model>
