<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| date input -> risk output | Wrong date math can mark expired items safe or defer action. |
| profile override -> category defaults | Overrides can accidentally mutate defaults used by other products. |
| domain result -> future task engine | Future task generation will trust state, command, and reasons. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Tampering | Risk window calculation | mitigate | Boundary tests pin 60/15/3/0 outputs and expired precedence. |
| T-02-05 | Repudiation | Reason output | mitigate | Return structured reason codes so future audit/task layers can explain why an action was requested. |
| T-02-06 | Denial of Service | Date parsing | mitigate | Keep parser narrow and deterministic; reject missing required date inputs as `uncertain` instead of throwing in normal calculation. |
</threat_model>
