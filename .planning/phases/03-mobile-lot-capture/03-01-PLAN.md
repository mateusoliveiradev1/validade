<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mobile form or barcode text -> capture schema | All entered values are untrusted until runtime validation succeeds. |
| capture repository -> SQLite | Invalid values or string interpolation could corrupt the local ledger. |
| correction request -> observation history | A correction could silently overwrite the fact it is meant to clarify. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Capture input schemas | mitigate | Use mode-discriminated Zod parsing and tests for malformed product, lot, location, and correction inputs. |
| T-03-02 | Tampering | SQLite adapter | mitigate | Use parameterized statements, deterministic migrations, and repository tests for latest snapshot behavior. |
| T-03-03 | Repudiation | Observation corrections | mitigate | Retain append-only observations and require a correction reason. |
</threat_model>
