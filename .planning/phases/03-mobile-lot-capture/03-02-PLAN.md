<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| manual search/code -> product selection | Lookup text can be incorrect and must not become an implicit registration. |
| product confirmation -> lot form | A category/profile mismatch could create an invalid date form. |
| form state -> durable repository | Stale or reused lot fields could be saved without a conscious collaborator action. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03 | Spoofing | Product discovery | mitigate | Require an explicit confirmation card before the lot form and test unknown-code behavior. |
| T-03-04 | Tampering | Lot form | mitigate | Validate each mode-specific date and location through contracts before local save. |
| T-03-05 | Repudiation | Repeat registration | mitigate | Reset all lot facts and store an actor label for each save. |
</threat_model>
