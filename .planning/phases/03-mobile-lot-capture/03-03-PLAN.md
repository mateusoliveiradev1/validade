<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| latest snapshot -> recent list | A derived current state could hide missing or uncertain physical facts. |
| collaborator outcome -> observation append | A high-impact status can be selected accidentally or without the necessary details. |
| correction form -> historical ledger | A correction must not erase or misrepresent the original floor observation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | Current lot snapshot | mitigate | Derive the list from append-only observations and test search/filter/state refresh. |
| T-03-06 | Repudiation | Consequential outcomes | mitigate | Require action-specific summary confirmation and persist actor/time/location/quantity state. |
| T-03-07 | Tampering | Correction path | mitigate | Require a correction reason and prove original observation history is retained. |
</threat_model>
