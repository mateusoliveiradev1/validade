<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| current lot snapshot -> task candidate | A stale or malformed snapshot could create the wrong operational work. |
| task contracts -> local task storage | Invalid status, due, or resolution fields could hide critical risk. |
| repeated refresh -> active task table | Duplicate tasks could confuse the shift and make safety state unreliable. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | Task derivation | mitigate | Use pure functions and tests for actionable-state filtering, priority, and compatible resolutions. |
| T-04-02 | Tampering | Task repository | mitigate | Validate task records at the boundary and use a stable active key to prevent duplicate local tasks. |
| T-04-03 | Repudiation | Task lifecycle | mitigate | Preserve created/updated/resolved timestamps and source risk reason codes in local records. |
</threat_model>
