<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| selected action -> task state | A wrong action could falsely resolve a critical risk. |
| destructive confirmation -> local observation/task resolution | A single tap could record withdrawal/loss without enough context. |
| evidence prompt -> future audit expectations | Evidence metadata could be mistaken for stored/private photo evidence. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-04-08 | Repudiation | Reinforced confirmation | mitigate | Show product, lot, location, quantity, chosen action, consequence, actor label, and timestamp before persistence. |
| T-04-09 | Information disclosure | Evidence prompt | mitigate | Store only placeholder/no-photo metadata and avoid real photos, binary data, object keys, or remote storage. |
