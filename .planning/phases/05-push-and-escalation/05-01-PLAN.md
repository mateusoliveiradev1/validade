<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| task state -> alert cadence | A stale or partially resolved task could keep notifying incorrectly or stop notifying too early. |
| alert content -> lock screen | Sensitive lot/detail context could leak before app unlock. |
| provider result -> task state | Delivery success could be mistaken for physical resolution. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | Alert cadence policy | mitigate | Pure tests assert exact cadence and recheck restart rules. |
| T-05-02 | Information disclosure | Notification content | mitigate | Contract/domain tests reject lot/detail leakage from lock-screen content. |
| T-05-03 | Repudiation | Leadership acknowledgement | mitigate | Acknowledgement has timestamped state but never mutates task `status` to resolved. |
</threat_model>
