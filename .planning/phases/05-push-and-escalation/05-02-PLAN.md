<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| native notification API -> app state | Permission/token/schedule results are external and unreliable. |
| alert state -> task repository | Alert failure or acknowledgement must not hide active tasks. |
| notification payload -> task open | Payload may be stale, resolved, or missing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Spoofing | Push-open payload | mitigate | Resolve payload by task id plus active key before opening the task. |
| T-05-05 | Denial of Service | Duplicate reminders | mitigate | Persist alert state and attempts, and make refresh idempotent. |
| T-05-06 | Information disclosure | Device token fixtures/logs | mitigate | Use fake tokens only and do not expose raw tokens in UI/tests/docs. |
</threat_model>
