<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| alert UI -> task UI | Notification state could crowd out or hide the physical task list. |
| leadership acknowledgement -> resolution | Receipt could be misread as task completion. |
| push payload -> navigation | A stale payload could open wrong task context. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-09 | Denial of Service | Hoje task list | mitigate | Tests assert channel failures keep task list visible and usable. |
| T-05-10 | Repudiation | Leadership receipt | mitigate | Receipt has timestamp/copy and leaves task active. |
| T-05-11 | Spoofing | Push tap routing | mitigate | Resolve by task id plus active key and show fallback on mismatch. |
</threat_model>
