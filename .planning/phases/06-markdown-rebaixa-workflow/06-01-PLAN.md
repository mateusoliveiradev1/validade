<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| risk/presence facts -> markdown request | A stale or uncertain lot could be marked for markdown without fresh physical confirmation. |
| stage command -> workflow state | A user action could skip approval, application, or final shelf confirmation. |
| evidence metadata -> future storage expectations | Placeholder evidence could be confused with real stored private photos. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | Markdown presence gate | mitigate | Pure helper and tests block stale/missing/not-present states before request or stage advance. |
| T-06-02 | Elevation of privilege | Approval contract | mitigate | Approval/rejection commands are separate, rejection requires reason, and early exceptions still require leadership approval. |
| T-06-03 | Information disclosure | Evidence contracts | mitigate | Schemas and tests reject URI/base64/object-key/binary fields; only placeholder/no-photo metadata is allowed. |
</threat_model>
