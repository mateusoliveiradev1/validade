<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repository command -> local workflow/task state | A partial transition could close one task without creating the next required task. |
| local storage -> evidence scope | SQLite could accidentally persist private photo data before Phase 8 storage controls. |
| alert acknowledgement -> stage completion | Leadership acknowledgement could be mistaken for approval/application/final confirmation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Tampering | Stage transitions | mitigate | Use transactions and tests asserting old task closure plus next task creation together. |
| T-06-05 | Information disclosure | SQLite evidence metadata | mitigate | Store only strict evidence JSON and source-test absence of URI/base64/object-key/R2 fields. |
| T-06-06 | Repudiation | Stage history | mitigate | Append actor/time/action/evidence history for each workflow transition. |
| T-06-07 | Elevation of privilege | Alert acknowledgement | mitigate | Tests prove acknowledgement never resolves a Today task or workflow stage. |
</threat_model>
