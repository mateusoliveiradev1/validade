<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| lot detail -> request command | Secondary entry could bypass presence or duplicate an active workflow. |
| UI evidence controls -> command payload | UI could submit generic text or binary-like evidence rather than structured metadata. |
| approval/rejection UI -> workflow state | A leadership decision could be submitted without explicit rejection reason. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-08 | Tampering | Lot detail markdown entry | mitigate | Entry state blocks request and routes to observation when presence is unsafe. |
| T-06-09 | Repudiation | Rejection panel | mitigate | Rejection reason is required before submit and is stored in workflow history. |
| T-06-10 | Information disclosure | Evidence UI | mitigate | UI submits only placeholder/no-photo metadata and tests reject URI/base64/object-key payloads. |
</threat_model>
