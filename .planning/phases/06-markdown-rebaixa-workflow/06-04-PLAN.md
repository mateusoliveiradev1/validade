## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| delayed stage UI -> operator action | A delayed stage could look resolved or low priority when product is still unconfirmed. |
| alert acknowledgement -> physical completion | Receiving a reminder could be mistaken for stage execution. |
| docs/smoke -> stakeholder confidence | Passing tests could be overread as real camera/storage/ERP integration. |
