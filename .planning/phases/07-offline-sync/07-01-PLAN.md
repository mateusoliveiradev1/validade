<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| physical action -> local command | A task action could be treated as centrally confirmed before sync acknowledgement. |
| command payload -> persisted outbox/API | Malformed or duplicate payloads could be retried as distinct actions. |
| conflict result -> UI decision | A safety-critical conflict could be hidden or auto-resolved. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | Cache readiness | mitigate | Pure policy requires active tasks and required snippets, not broad local data. |
| T-07-02 | Spoofing | Safety verdict | mitigate | Pending critical/conflict/stale states force qualified safety copy. |
| T-07-03 | Repudiation | Idempotent command | mitigate | Every command has idempotency key and actor/time payload from existing command schemas. |
| T-07-04 | Information disclosure | Evidence/sync payload | mitigate | Strict schemas and tests reject raw URI/base64/object key/image fields. |
</threat_model>
