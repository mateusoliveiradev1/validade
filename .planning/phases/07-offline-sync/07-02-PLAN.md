<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UI/task action -> repository outbox | A physical action could be lost if projection happens before command persistence. |
| local pending projection -> safety verdict | A local action could hide expired/critical risk before central acknowledgement. |
| retry -> command table | Retrying could duplicate a physical action if idempotency is unstable. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Tampering | Offline command write | mitigate | Transaction inserts command before local projection; tests assert ordering outcome. |
| T-07-06 | Repudiation | Command history | mitigate | One row per physical command with actor/time/payload and no destructive grouping. |
| T-07-07 | Spoofing | Pending central acknowledgement | mitigate | Task sync metadata distinguishes local save from synced ack. |
| T-07-08 | Denial of Service | Retry/idempotency | mitigate | Unique idempotency key and retry state reuse same command row. |
</threat_model>
