<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| network state -> sync attempt | Connectivity can be unknown or stale and must not imply successful sync. |
| mobile outbox -> API endpoint | Remote seam must parse strict commands and never accept malformed/private payloads. |
| duplicate retry -> central command service | Duplicate idempotency keys must not create duplicate central effects. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-09 | Denial of Service | Sync runner | mitigate | Limited retry/backoff and visible failure state. |
| T-07-10 | Spoofing | Network state | mitigate | NetInfo is only a trigger; transport ack is required for synced state. |
| T-07-11 | Tampering | API sync endpoint | mitigate | Strict batch parsing and 400 on malformed input. |
| T-07-12 | Repudiation | Idempotency service | mitigate | Duplicate idempotency keys return existing outcome, not another action. |
</threat_model>
