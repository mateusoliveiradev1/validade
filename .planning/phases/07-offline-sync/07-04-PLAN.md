<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local save feedback -> operator interpretation | User could read local save as system confirmation. |
| conflict panel -> destructive discard | User could discard critical physical work without accountability. |
| cached startup -> safety verdict | Stale cached tasks could be read as current central safety. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-07-14 | Tampering | Conflict resolution | mitigate | Critical discard requires confirmation plus reason. |
| T-07-15 | Repudiation | Task panel local save | mitigate | Offline command records actor/time/payload before feedback. |
| T-07-16 | Information disclosure | UI/docs/smoke | mitigate | Tests and docs use fictitious data; no raw photo/storage payloads. |
</threat_model>
