<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| request identity -> AuthProvider | Tokens/sessions may be forged, expired, or malformed. |
| verified subject -> membership store | JWT claims must not become operational role/store authority. |
| authorized actor -> resource repository | Known IDs must not bypass store scoping. |
| schema migration -> Neon | Schema changes must be proven on a temporary branch without exposing credentials. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T8-01 | Elevation/Information disclosure | Store-scoped API | block | Capability plus `store_id` in every repository operation and negative IDOR tests. |
| T8-02 | Spoofing | Actor context | block | AuthProvider verifies identity; membership store supplies role/store; body claims ignored. |
| T8-03 | Elevation | Admin operations | block | Operational lead capabilities require explicit lead membership in the target store. |
| T8-04 | Tampering | Audit schema | block | Append-only database guard and no update/delete application adapter. |
| T8-05 | Repudiation | Writes/events | block | Unique idempotency keys and actor/role snapshots. |
| T8-10 | Repudiation | Denied access | mitigate | Sanitized denial event without secrets or target disclosure. |
</threat_model>
