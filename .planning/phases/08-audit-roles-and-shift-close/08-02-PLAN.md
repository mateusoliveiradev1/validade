<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| domain mutation -> audit append | A successful mutation could lack or diverge from its event. |
| offline event -> central acknowledgement | Replay could duplicate or rewrite history. |
| audit repository -> query projection | Raw metadata or another store's records could leak. |
| query UI -> privileged evidence/diagnostics | A visible link could bypass authorization. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T8-01 | Information disclosure | Audit query | block | Store predicate in repository operation plus negative IDOR tests. |
| T8-04 | Tampering | Audit ledger | block | Append-only repository/DB and linked correction events. |
| T8-05 | Repudiation | Replay | block | Unique idempotency and one mutation+event transaction. |
| T8-08 | Information disclosure | Projection/UI | mitigate | Allowlisted operational projection; no raw metadata, URLs, headers, or payloads. |
| T8-10 | Repudiation | Denied access | mitigate | Sanitized denial events produced by Plan 01 and queryable only by authorized roles. |
</threat_model>
