<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| device file -> upload queue | A local URI or bytes could leak into command sync/logs. |
| upload request -> API/R2 | Malicious type/size/hash/store/target could be accepted. |
| private object -> authorized viewer | Object key or bearer URL could leak or cross stores. |
| lifecycle deletion -> audit metadata | Binary expiry could erase accountability. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T8-01 | Information disclosure | Evidence access | block | Capability plus store-scoped repository lookup before get/put/invalidate. |
| T8-06 | Information disclosure | R2 object | block | Private bucket, server-generated key, authorized streaming, no persistent signed URL. |
| T8-07 | Denial/Tampering | Upload | block | MIME/size/hash validation, streaming limit, retry idempotency, fake adapter tests. |
| T8-08 | Information disclosure | Logs/UI | mitigate | Redaction and source assertions banning key/query URL/raw headers. |
| T8-05 | Repudiation | Upload retry | block | Idempotency and separate upload lifecycle events. |
</threat_model>
