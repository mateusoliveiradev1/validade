<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local Today state -> safe claim | Cached/offline state may be stale or incomplete. |
| lead confirmation -> central transaction | State can change between review and commit. |
| unsafe close -> enforcement/handoff | Closing could accidentally silence pending tasks or imply receipt/resolution. |
| reopen -> historical snapshot | A correction could overwrite the original close. |
