<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| invite link -> activation | Tokens can leak, expire, be replayed, or be guessed. |
| password form -> credential verifier | Passwords must not be stored or logged raw. |
| session token -> capability context | A valid token does not imply still-active membership. |
| privacy request -> public repo/logs | Requests can contain personal text and must not leak to docs/tests/logs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T9-01 | Spoofing | AuthProvider/session | block | Hashed session lookup plus server-owned membership/capability refresh. |
| T9-02 | Tampering | Invite activation | block | Hashed one-time invite token, expiry, status transition, idempotency. |
| T9-03 | Repudiation | Auth/admin actions | mitigate | Append sanitized audit for invite, blocked/revoked access, recovery, privacy request intake. |
| T9-04 | Information disclosure | Recovery/invite errors | block | Generic recovery response and invalid invite copy with no account existence disclosure. |
| T9-05 | Elevation | Client role/store claims | block | No client auth request can grant role/store; AuthorizationService remains source of truth. |
</threat_model>
