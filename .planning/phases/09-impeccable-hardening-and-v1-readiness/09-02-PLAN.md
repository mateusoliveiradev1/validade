<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| launch/loading -> user trust | Splash/loading could imply session or safety before verification. |
| auth gate -> CaptureApp | UI bypass could expose operational flows without session context. |
| local action -> central truth | Polish could accidentally relabel local/offline/evidence states as confirmed. |
| generated assets -> public repo | Icon/splash generation must not embed real data or secret metadata. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T9-06 | Spoofing | Mobile shell | block | AuthGate controls CaptureApp render and consumes server session context. |
| T9-07 | Information disclosure | Auth errors | mitigate | Generic invalid invite/recovery/session states; no account existence disclosure. |
| T9-08 | Repudiation | Offline/evidence UI | block | Existing pending labels stay inline until central acknowledgement. |
| T9-09 | Tampering | App assets/config | mitigate | Deterministic asset script and security scans forbid real/private artifacts. |
</threat_model>
