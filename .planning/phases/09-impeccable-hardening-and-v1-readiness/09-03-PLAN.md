<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| session shell -> web surfaces | Hidden UI is not authorization; API must still enforce scope. |
| command-center projection -> leadership decisions | Unsupported or stale metrics could mislead operations. |
| admin invite/revoke -> access control | UI mistakes could grant or revoke wrong person/store/role. |
| evidence access -> private storage | Web must not reveal keys or signed URLs before authorization. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T9-10 | Information disclosure | Command Center | block | Projection uses store-scoped existing records and no unsupported sales/BI data. |
| T9-11 | Elevation | Admin invite/membership | block | Server-owned user.manage guard, explicit consequence confirmation, audit event. |
| T9-12 | Spoofing | Web session | block | Shell consumes authenticated session contract and logout/session-expired states. |
| T9-13 | Disclosure | Evidence access | block | Existing evidence confirmation retained; no keys/signed URLs in UI/tests. |
</threat_model>
