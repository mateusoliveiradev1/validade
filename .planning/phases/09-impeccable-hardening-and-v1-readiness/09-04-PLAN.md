<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| fake fixtures -> release confidence | Deterministic data must cover real journey semantics without pretending to be provider evidence. |
| UI scanner -> source tree | Static scan can miss runtime-only defects and must complement E2E/manual checks. |
| performance script -> secrets/logs | Measurements must not leak provider URLs, tokens, device paths, or evidence IDs. |
| E2E happy path -> denied path | Release evidence must include both permitted and blocked operations. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T9-14 | Spoofing/Elevation | E2E auth/admin | block | Include invite/auth/session/no-permission/forged role and cross-store denial journeys. |
| T9-15 | Repudiation | Evidence/sync/shift tests | block | E2E proves local vs central ack, evidence ack, unsafe/safe close, and audit trail. |
| T9-16 | Information disclosure | Fixtures/logs | block | Static scans and fictional fixtures reject secrets, signed URLs, build URLs, and real data. |
| T9-17 | Denial of service | Performance | mitigate | Budgets check load/filter/sync paths and document environment variance. |
</threat_model>
