<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| release docs -> operator action | Misleading install/admin docs could expose secrets or unsafe access. |
| manual provider check -> automated release claim | Missing device/EAS checks must not be reported as passed. |
| residual risk -> release decision | Critical auth/privacy/security/flow risks cannot be accepted as ordinary risk. |
| APK/build output -> public repo | Build URLs, tokens, and binaries with private metadata must not be committed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T9-18 | Information disclosure | Release docs/APK | block | Security scans and doc review forbid secrets, build URLs, object keys, real data. |
| T9-19 | Repudiation | Go/no-go record | mitigate | Command results, manual blockers, and validation status recorded in release docs and summary. |
| T9-20 | Elevation | Account operations docs | block | Runbook states server-owned session/membership authority and revocation refresh behavior. |
| T9-21 | Spoofing | Manual provider evidence | mitigate | Unrun provider/device checks remain blockers unless exact local equivalent passes. |
</threat_model>
