<threat_model>
## Trust Boundaries

| Boundary | Risk | Mitigation |
|---|---|---|
| mobile session -> prepare-turn | Client may request another store. | Resolve store through session/membership and deny cross-store ids. |
| central package -> SQLite cache | Cache may be stale or partial. | Persist freshness and show stale/unavailable labels. |
| empty central tables -> UI verdict | Empty rows can be misread as safe. | Use `needs_review`/explicit no-data copy until central projection is current. |
| sync transport -> API | Query store id can be spoofed. | Bind sync and prepare-turn to bearer/session scope. |
</threat_model>
