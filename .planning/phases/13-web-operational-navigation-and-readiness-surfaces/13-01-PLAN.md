<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Session context to web route access | Browser receives session actions and decides which route content to render. Server remains the authority, but UI must fail closed. |
| Web route to `/command-center` | Shared host fetches store-scoped projection using authenticated fetcher. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Elevation of privilege | `routeAllowed`, `firstAllowedRoute`, `navItemState` | mitigate | Keep all operational routes behind `canReadCommandCenter`; do not infer permission from role label or route name. |
| T-13-02 | Tampering | Route-specific projection helpers | mitigate | Helpers must only derive display summaries from `CommandCenterProjection`; they must not synthesize safe states when projection data is missing. |
| T-13-SC | Tampering | npm package surface | mitigate | Do not add new packages for routing or UI blocks in this plan. If a package is proposed, stop for review before install. |
</threat_model>
