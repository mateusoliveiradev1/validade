<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Device build metadata to update UI | Public build labels from projection become web-visible update status. |
| Optional public update config to clickable UI | Any configured APK/QR path could accidentally expose a private build URL or token. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-06 | Information disclosure | Atualizacoes safe update path | mitigate | Render links only after public-safe validation; deny token/private/build URL patterns and default to manual instructions. |
| T-13-11 | Spoofing | Installed-build compatibility display | mitigate | Compare/display only against approved artifact fields from projection; do not mark a device current from repo version alone. |
| T-13-SC | Tampering | npm package surface | mitigate | Do not add QR/link generation dependencies in this plan; use text/manual fallback unless a safe existing primitive is enough. |
</threat_model>
