## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Native/app metadata to public UI | Build info may originate from native modules or Expo config and must stay public-safe. |
| Auth/session state to controls | Account/store/role display must remain read-only and server-owned. |
| Sign-out to local work | Ending a session must not mutate pending commands, conflicts, or tasks. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-14-08 | Elevation of privilege | Account/store controls | mitigate | Render read-only session facts; do not add store/role switching or capability changes. |
| T-14-09 | Tampering | Sign-out confirmation | mitigate | Test logout does not resolve tasks, sync conflicts, or clear pending outbox state. |
| T-14-SC | Tampering | npm package surface | mitigate | Do not add update, linking, deep-link, QR, or account-management packages in this plan. |
</threat_model>
