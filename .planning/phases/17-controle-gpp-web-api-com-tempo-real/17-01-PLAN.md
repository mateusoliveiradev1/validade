<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Web session to backend capability | The browser can request routes, but backend derives all GPP authority from session membership and feature flag. |
| Feature flag to tested build 170 | Hidden GPP code must not alter the current mobile validation flow or default web route for non-GPP users. |
| Contract payload to central writes | GPP requests must be strict and unable to claim central success or actor authority. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Elevation of privilege | GPP role/capability matrix | mitigate | Dedicated `gpp` role with allow/deny tests against lead/admin/governance capabilities. |
| T-17-02 | Spoofing | Session actions and feature flag | mitigate | Actions and flag derive on server; contract normalizes missing values to false. |
| T-17-03 | Tampering | GPP request contracts | mitigate | Strict schemas reject forged central status, unknown fields, missing code, and invalid status transitions. |
</threat_model>
