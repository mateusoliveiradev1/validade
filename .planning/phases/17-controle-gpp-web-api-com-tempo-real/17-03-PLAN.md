<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser request to API route | Client can submit payloads, but backend owns store, role, status, and creator authorization. |
| Repository mutation to audit event | A central mutation must not be considered complete without audit unless the API returns failure/retry. |
| Central commit to realtime publish | Events are emitted only after central success and never substitute for central read truth. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-05 | Elevation of privilege | GPP API routes | mitigate | Capability checks, creator/status checks, store-scope checks, denial tests. |
| T-17-06 | Repudiation | GPP audit | mitigate | Append audit for every non-replayed mutation with previous/next state and idempotency key. |
| T-17-07 | Tampering | Realtime publish | mitigate | Publish only after central success; payload is refresh hint only. |
</threat_model>
