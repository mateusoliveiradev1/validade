<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WebSocket connection to store room | Connection must be authorized and store-scoped before receiving events. |
| Event hint to visible state | Event delivery may trigger refresh, but cannot directly mutate business truth. |
| Realtime transport to central persistence | Transport failure must not roll back successful central DB writes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-07 | Spoofing | WebSocket room | mitigate | Session authorization and store-scoped room ids before upgrade. |
| T-17-08 | Tampering | Event payload | mitigate | Contract rejects authoritative payloads; web re-reads central snapshot. |
| T-17-09 | Denial of service | Realtime dependency | mitigate | No-op publisher, paused state, manual refresh, and bounded polling fallback. |
</threat_model>
