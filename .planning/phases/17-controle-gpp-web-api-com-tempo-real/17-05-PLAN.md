<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Session actions to route visibility | Web hiding is not authorization, but must avoid exposing confusing broken shells. |
| Central mutation response to UI success | UI can only show success after API resolves central-confirmed response. |
| Realtime event to rendered rows | Event is a refresh hint and cannot directly update business state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-09 | Information disclosure | Web GPP route | mitigate | Feature/capability gate and no route shell for unauthorized users. |
| T-17-10 | Tampering | Web mutation state | mitigate | Disable during save, parse responses, show central failure, keep retryable rows. |
| T-17-11 | Spoofing | Realtime UI updates | mitigate | Re-read central snapshot on event; event payload cannot mark success. |
</threat_model>
