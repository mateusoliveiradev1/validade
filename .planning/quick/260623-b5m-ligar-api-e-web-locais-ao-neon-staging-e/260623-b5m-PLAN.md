# Quick Task 260623-b5m: Staging Neon e autenticacao persistente

## Objetivo

Ligar a API e o painel web locais ao banco Neon de staging, sem permitir fallback inseguro para memoria e sem versionar segredos.

## Execucao

1. Fazer o Worker exigir URL do banco e peppers de autenticacao para compor a API persistente.
2. Encaminhar as rotas operacionais e de autenticacao do Vite para a API local.
3. Validar os testes da API e do web, depois confirmar um login real contra a branch staging.
4. Corrigir o fluxo de ativação para mostrar erros acionáveis em vez de falhar silenciosamente.
5. Alinhar a política de senha com a credencial de staging solicitada para o admin.
6. Recriar o Centro de Privacidade com hierarquia, shell e leitura mobile/desktop de produto.

## Seguranca

- Connection string e peppers ficam somente em configuracao local ignorada e em segredos do ambiente de deploy.
- A producao nao e alterada; o banco alvo e a branch Neon `staging`.
