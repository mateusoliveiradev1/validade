# Quick Task 260623-h7v: Polir UI/UX mobile e preparar APK staging

## Goal

Melhorar a percepcao de produto final no app Android, especialmente login, shell autenticado e tela Hoje, usando o contrato Impeccable/Phase 09 como guia, e validar o caminho realista para gerar um APK staging instalavel em dispositivo fisico.

## Scope

1. Reforcar identidade operacional no login sem transformar a tela em marketing.
2. Reduzir a sensacao de cards iguais na tela Hoje e priorizar proximo passo, seguranca da area de venda, sync e fechamento.
3. Melhorar shell autenticado para ficar mais compacto e confiavel no topo.
4. Checar EAS/APK staging sem versionar tokens, build URLs, secrets ou dados reais.
5. Testar no emulador e rodar gates focados antes de qualquer handoff.

## Constraints

- Sem segredos, links privados de build, tokens Expo, connection strings ou dados reais no Git.
- APK externo so e valido se apontar para uma API staging publica; localhost serve apenas para emulador/desenvolvimento.
- UI deve preservar os termos semanticamente verdadeiros do Phase 09: seguro, sincronizado, confirmado e pendente so quando o dominio sustenta.
