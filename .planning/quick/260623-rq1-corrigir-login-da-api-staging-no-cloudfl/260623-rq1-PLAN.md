# Quick Task 260623-rq1: Corrigir login staging no Cloudflare

## Goal

Corrigir o `500` no login da API staging publicada no Cloudflare Workers, mantendo a conta admin/líder utilizável para teste real no APK staging.

## Findings

- `/auth/login` com conta inexistente retorna `401`, então DB/rota estão acessíveis.
- `/auth/login` com conta existente retorna `500` no Worker publicado.
- O mesmo login com o mesmo banco e segredos funciona localmente.
- A diferença aponta para o custo PBKDF2 atual (`310000`) excedendo o limite prático de CPU do Worker gratuito.

## Tasks

1. Reduzir o algoritmo padrão de senha para um custo compatível com o Worker staging.
2. Fazer o verificador rejeitar hashes com iterações acima do limite suportado, evitando `500` por CPU.
3. Atualizar testes do repositório de auth.
4. Resetar a credencial staging da conta solicitada para o novo marcador de algoritmo.
5. Redeployar API staging e validar login real sem imprimir token/senha.

## Verification

- Testes de `@validade-zero/database` passam.
- API staging retorna `200` para login da conta admin/líder.
- `/auth/session` ou payload de login confirma `lead` e `canManageUsers=true`.
