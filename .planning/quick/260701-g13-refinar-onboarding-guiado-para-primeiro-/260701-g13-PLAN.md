# Refinar onboarding guiado para primeiro turno assistido persistente

## Objetivo

Transformar o guia atual em um onboarding operacional de primeira execucao: aparecer antes de Hoje apenas quando o usuario ainda nao chegou ao primeiro valor real, nao repetir para quem ja operou, e sobreviver a reinstalacao quando a API central estiver disponivel.

## Decisoes

- O fluxo se chama `Primeiro turno assistido`.
- O momento de valor e registrar um lote fisico real e voltar para Hoje com o resultado visivel.
- `Hoje` nao deve carregar um card de onboarding permanente.
- Usuarios ja ativados por fatos reais entram direto na operacao.
- Conclusao/pulo do onboarding usa `flowId + version + subjectId + storeId`.
- Persistencia local e obrigatoria; persistencia central e usada quando houver API.
- Reinstalacao sem internet pode mostrar uma versao curta com `Pular e abrir Hoje`, mas nao deve bloquear operacao.

## Verificacao

- Testes de contrato/API para leitura e gravacao do progresso central.
- Testes mobile para:
  - novo usuario antes de Hoje;
  - usuario ja ativado sem repeticao;
  - pulo/conclusao persistidos localmente;
  - guia reabrivel por Ajustes/acao secundaria.
- `pnpm.cmd check`.
